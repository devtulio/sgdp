1. Visão Geral do Sistema

1.1 Objetivo

O SGDP (Sistema de Gestão de Documentos da Procuradoria) é uma aplicação web de uso interno desenvolvida para a Procuradoria-Geral municipal. Seu propósito é centralizar e controlar os atos jurídicos produzidos pelo órgão — substituindo planilhas e pastas físicas por um repositório digital consultável, rastreável e com acesso simultâneo para todos os procuradores.

1.2 Tipos de documento gerenciados

TipoDescriçãoNumeração
LeiAto normativo aprovado pela Câmara Municipalnº 001/2026…
DecretoAto normativo do Executivo Municipalnº 001/2026…
PortariaAto administrativo interno do Executivonº 001/2026…
ParecerManifestação jurídica da Procuradorianº 001/2026…
OfícioComunicação oficial externa da Procuradorianº 001/2026…

Cada tipo possui sua própria sequência numérica independente, reiniciada a cada ano.

1.3 Arquitetura simplificada

Um único computador executa o servidor (server.py) e armazena todos os dados (banco SQLite + PDFs em disco). Os procuradores acessam pelo navegador — seja no próprio computador servidor (localhost:3001) ou em outros computadores da mesma rede (192.168.x.x:3001). Não é necessária conexão com a internet.

Nota: O servidor não encerra sozinho ao fechar as janelas do navegador — ele continua rodando até ser interrompido manualmente (Ctrl+C na janela do terminal, ou fechando essa janela) ou o computador ser desligado. Assim que a última sessão ativa termina, o sistema apenas dispara um backup automático em segundo plano, sem encerrar.

2. Instalação e Inicialização

2.1 Requisitos

  • Python 3.8 ou superior — disponível em python.org; marcar "Add Python to PATH" na instalação
  • Google Chrome ou Microsoft Edge atualizado
  • Windows 10 ou 11
  • Nenhuma dependência externa — apenas a biblioteca padrão do Python

2.2 Primeiros passos

  1. Copie a pasta SGDP/ para o computador que atuará como servidor (preferencialmente um computador sempre ligado durante o expediente)
  2. Clique duas vezes em Iniciar SGDP.bat
  3. Um menu aparece no terminal com duas opções: [1] Diagnóstico ou [2] Iniciar Servidor — escolha 2 (ou apenas aguarde: o padrão é iniciar o servidor)
  4. O navegador abrirá automaticamente na tela de login
  5. Use as credenciais iniciais: usuário admin / senha admin123
  6. O sistema exige a troca de senha no primeiro acesso — não é possível continuar sem definir uma senha nova
  7. Cadastre os demais procuradores na tela de Usuários
Importante: não exclua o arquivo sgdp.db nem a pasta uploads/ — eles contêm todos os dados e PDFs do sistema. Realize backups periódicos pela tela de Backup.

2.3 Acesso em rede local

O SGDP foi projetado para uso multiusuário em rede local (LAN): uma única máquina executa o servidor (e guarda o banco de dados sgdp.db) e os demais procuradores acessam pelo navegador, sem instalar nada.

Na máquina servidora (configuração única):

  1. Execute o arquivo Liberar Porta SGDP.bat como Administrador (botão direito → Executar como administrador). Ele cria a regra no Firewall do Windows liberando a porta 3001 para conexões de entrada — precisa ser feito apenas uma vez
  2. Inicie o sistema normalmente pelo Iniciar SGDP.bat e deixe a máquina ligada. Ao iniciar, o console exibe o endereço de rede pronto para distribuir aos procuradores (ex.: Rede: http://192.168.1.10:3001/SGDP.html)

Nos computadores dos procuradores: basta abrir o navegador (Chrome ou Edge) no endereço do servidor — cada um faz login com sua própria conta, e todos trabalham sobre os mesmos dados.

Conexão não funciona? Execute o Diagnostico SGDP.bat (ou escolha a opção [1] Diagnóstico no menu que aparece ao abrir o Iniciar SGDP.bat) na máquina servidora: ele descobre o IP, verifica a porta 3001, as regras de firewall e a acessibilidade pela rede, com instruções de correção item a item.
Dica: Crie um atalho no navegador de cada procurador apontando para o endereço IP do servidor para facilitar o acesso diário.
Uso restrito à rede interna. A comunicação entre as máquinas é HTTP simples (sem criptografia de transporte) — adequado para uma rede interna confiável do órgão, mas nunca exponha a porta do sistema à internet (redirecionamento de porta no roteador, DMZ etc.). Para acesso remoto, utilize a VPN institucional.

3. Login e Perfis de Acesso

O SGDP possui dois perfis de acesso:

PerfilPermissões
Administrador Acesso total: documentos, usuários, auditoria e backup
Procurador (padrão) Criar, editar e excluir documentos; upload e download de PDFs; dashboard

As sessões de login têm duração de 8 horas. Ao expirar, o sistema retorna automaticamente à tela de login. O logout manual encerra a sessão imediatamente.

Além do perfil, cada usuário pertence a um Departamento fixo — Procuradoria-Geral ou Gabinete — definido pelo administrador ao criar ou editar o usuário. O departamento determina quem pode editar documentos não-sigilosos de terceiros (ver 4.6).

Segurança: as senhas são armazenadas com criptografia PBKDF2-HMAC-SHA256 com salt individual por usuário. Não é possível recuperar a senha esquecida — apenas um administrador pode redefini-la pela tela de Usuários.

4. Cadastro de Documentos

Para cadastrar um novo documento, clique no item correspondente na barra lateral (ex: Leis) e em seguida no botão + Nova Lei no canto superior direito.

4.1 Campos do formulário

CampoObrigatórioDescrição
TipoSimDefinido pelo botão clicado; não pode ser alterado após salvar
NúmeroNãoPreenchido automaticamente com o próximo número disponível; pode ser ajustado
AnoSimPreenchido com o ano atual; editável
DataSimData do ato (assinatura, publicação ou expedição)
EmentaSimDescrição resumida do conteúdo do documento
AssuntoSimCategoria do documento, selecionada em lista pré-definida (ver 4.4)
Partes / EnvolvidosNãoNomes das partes, requerentes, destinatários ou interessados
ObservaçõesNãoInformações internas adicionais; não aparecem em relatórios
🔒 SigilosoNãoRestringe a visibilidade do documento apenas a quem o criou e a administradores (ver 4.6)
PDF AssinadoNãoArquivo PDF do documento já assinado digitalmente

4.2 Numeração automática

Ao abrir o formulário de novo documento, o campo Número aparece em branco com a indicação "auto". O sistema atribuirá o próximo número disponível para aquele tipo e ano ao salvar. Para definir um número específico (ex: ao cadastrar documentos retroativos), basta digitar o valor desejado antes de salvar.

Nota: a numeração é por tipo + ano. Leis e Decretos têm sequências independentes entre si. A sequência reinicia do 1 a cada início de ano.

4.3 Anexar PDF assinado

O fluxo recomendado para cada documento é:

  1. O procurador elabora o documento em Word (.docx)
  2. Exporta para PDF
  3. Realiza a assinatura digital no PDF (via GOV.BR, ICP-Brasil ou assinatura eletrônica municipal)
  4. Acessa o SGDP e cadastra ou edita o documento
  5. Faz o upload do PDF assinado na seção "PDF Assinado" do formulário

O PDF pode ser anexado no momento do cadastro ou posteriormente, editando o documento. Para substituir um PDF já anexado, clique em ↺ Substituir. Para visualizar, clique em 👁 Ver; para baixar, em ⬇.

Na tela de visualização do PDF há também os botões 🖨 Imprimir / Salvar PDF (imprime ou salva uma cópia do documento exibido) e ✉ Enviar por e-mail (envia o PDF assinado como anexo — requer SMTP configurado, ver 6.6).

Dica: você também pode arrastar o arquivo PDF diretamente para a área de upload em vez de clicar para selecionar.

4.4 Assunto e campos específicos por tipo

O campo Assunto é preenchido a partir de uma lista de categorias que muda conforme o tipo de documento: 14 categorias gerais (Administrativo Geral, Pessoal e RH, Contratos e Convênios etc.) para Lei, Decreto, Portaria e Ofício; e 46 categorias jurídicas específicas para Parecer, cobrindo temas de licitações, pessoal, tributário, urbanismo, meio ambiente e mais.

Alguns tipos de documento exibem campos adicionais, ativados automaticamente conforme o Assunto selecionado:

TipoCampoQuando aparece
Parecer PA (Processo Administrativo) Quando o Assunto pertence ao grupo de licitações (ex: "Dispensa de Licitação", "Pregão Eletrônico")
Modalidade / processo-filho (DL, PE, CE, IL, RP, AT, RJ, RE, PR…)
Portaria Tipo de ato (Nomeação, Exoneração, Designação…) Sempre, ao selecionar o tipo Portaria
Cargo
Nota: o vínculo PA + Modalidade conecta o Parecer ao processo correspondente no Departamento de Licitações — por exemplo, "PA 10/2026 DL 05/2026" identifica o Processo Administrativo 10/2026 e a Dispensa de Licitação 05/2026 vinculada a ele.

4.5 Importação em lote via CSV

Para cadastrar vários documentos de uma vez (ex: digitalizar um arquivo histórico), use o botão ⇪ Importar CSV disponível em cada tela de listagem de documentos.

  1. Prepare um arquivo .csv com cabeçalho na primeira linha e as colunas: numero, ano, data, ementa, partes, observacoes, assunto (o campo tipo é opcional — usa o tipo da tela atual se omitido)
  2. Clique em Importar CSV, selecione o arquivo e confira a prévia
  3. Clique em Importar — documentos sem numero informado recebem numeração automática

Ao final, o sistema exibe quantos documentos foram importados e lista erros por linha (ex: número já existente para aquele tipo/ano).

4.6 Documentos sigilosos e departamentos

Toda listagem de documentos (e o Dashboard) mostra a coluna Origem, com o departamento de quem criou o documento — útil pra identificar rapidamente se um Ofício, por exemplo, veio da Procuradoria-Geral ou do Gabinete.

Regras de acesso a um documento:

  • Edição/exclusão (documento não-sigiloso): qualquer usuário do mesmo departamento de quem criou pode editar, excluir ou assinar — além do administrador
  • Marcado como 🔒 Sigiloso: aparece na listagem, no dashboard, na busca e nos relatórios apenas para quem o criou e para administradores — mesmo colegas do mesmo departamento não veem
  • Alterar a marcação de sigilo: só quem criou o documento ou um administrador pode marcar/desmarcar como sigiloso — nem colegas de departamento podem
Nota: um administrador sempre vê e pode editar todos os documentos, sigilosos ou não, de qualquer departamento.

4.7 Assinatura digital, verificação e histórico

Além de anexar um PDF já assinado externamente (ver 4.3), o SGDP pode assinar digitalmente dentro do próprio sistema, usando um certificado ICP-Brasil A1 (arquivo .pfx):

  1. Na listagem do documento, clique no ícone 🔏 (disponível para quem pode editar o documento — ver 4.6)
  2. Selecione o arquivo do certificado (.pfx) e informe a senha
  3. Opcionalmente, envie um PDF novo para assinar — se nenhum for enviado, o sistema assina o PDF já anexado ao documento
  4. Confirme — o PDF assinado substitui o anexo do documento
Requisito: a assinatura ICP-Brasil depende do módulo opcional pyhanko (ver Requisitos, 2.1). Se não estiver instalado, o botão exibe uma mensagem orientando a instalação.

Toda assinatura feita pelo sistema gera um código de verificação curto (ex: 65C9-CA87), consultável sem login em /verificar/<código> — a página mostra o documento, o signatário, os dados do certificado e a data, permitindo que qualquer pessoa (ex: outro órgão, um cidadão) confirme a autenticidade sem precisar acessar o SGDP. Documentos assinados pelo sistema mostram o ícone 🔏 como link direto para essa página.

Toda edição de um documento também fica registrada em seu histórico de revisões, acessível pelo ícone 🕘 na listagem — mostra os campos como estavam antes de cada alteração, com autor e data/hora.

4.8 Etiquetas e vínculos entre documentos

Etiquetas (tags): no campo Etiquetas do formulário de documento, informe uma ou mais palavras separadas por vírgula (ex: urgente, folha de pagamento) — o campo sugere etiquetas já usadas anteriormente. Clique em uma etiqueta na listagem para filtrar por ela. O Relatório de Etiquetas (ver Seção 7) agrupa todos os documentos por etiqueta.

Vínculos entre documentos: pelo ícone 🔗 na listagem, é possível relacionar dois documentos com um tipo de vínculo:

TipoSentido inverso exibido no outro documento
RevogaRevogado por
AlteraAlterado por
ComplementaComplementado por
ReferenciaReferenciado por

Cada vínculo aparece nos dois documentos envolvidos, com o rótulo correspondente ao sentido (ex: a Lei que revoga mostra "Revoga", a Lei revogada mostra "Revogado por"). A partir do modal de Vínculos, o botão Gerar Cadeia Normativa monta uma árvore com todos os documentos conectados direta ou indiretamente ao documento atual (revogações, alterações, complementos e referências em cadeia), útil para entender rapidamente todo o histórico normativo de um tema.

5. Busca e Filtros

Em cada tela de listagem (Leis, Decretos etc.) há uma barra de ferramentas com:

  • Campo de busca: pesquisa por número, ementa e partes envolvidas. A busca em ementa/partes é por palavra (não apenas por trecho exato) — encontra o documento mesmo que as palavras buscadas não estejam juntas ou na mesma ordem do texto. Pressione Enter ou clique em Filtrar para aplicar.
  • Filtro por ano e filtro por etiqueta (ao clicar numa etiqueta de um documento): exibem apenas os documentos correspondentes.
  • Paginação: 50 documentos por página; navegue pelas setas ou números.
Exemplo: para localizar um parecer que menciona "João da Silva", acesse Pareceres, digite "João" no campo de busca e pressione Enter.

Além da busca por tela, o atalho Ctrl+K (ou o ícone de lupa na barra lateral) abre uma busca global que pesquisa em todos os tipos de documento ao mesmo tempo, de qualquer tela do sistema.

6. Administração

As telas de administração são visíveis apenas para usuários com perfil Administrador.

6.1 Gestão de usuários

Acesse Usuários na barra lateral para:

  • Criar usuário: clique em + Novo Usuário; informe nome completo, nome de login (sem espaços), senha (mínimo 6 caracteres), departamento (Procuradoria-Geral ou Gabinete — ver 4.6) e se é administrador. CPF, e-mail, cargo e matrícula são opcionais.
  • Editar: altere qualquer campo, inclusive senha, departamento e perfil. O nome de login não pode ser alterado após a criação.
  • Desativar: desmarque "Conta ativa" para bloquear o acesso sem excluir o histórico do usuário.
  • Excluir: remove o cadastro permanentemente. Não é possível excluir seu próprio usuário, nem um usuário que já criou, editou ou assinou algum documento (ou tem qualquer outra ação registrada na auditoria) — nesses casos, use Desativar para preservar o histórico.
Atenção: defina pelo menos um usuário administrador além do admin padrão. Se a senha do único administrador for perdida, não há como recuperá-la sem intervenção técnica direta no banco de dados.

6.2 Auditoria

A tela de Auditoria exibe um histórico completo de todas as ações realizadas no sistema, com data/hora, usuário responsável, tipo de ação e detalhes. É possível filtrar por usuário, tipo de ação e período, e exportar o resultado filtrado em CSV pelo botão correspondente na própria tela. As ações registradas são:

6.3 Backup e restauração

Acesse a aba Dados em Configurações. O sistema mantém dois tipos de backup, lado a lado, guardados na mesma pasta (configurável pelo botão Selecionar pasta):

TipoArquivoConteúdo
Sistema (JSON)SIS_SGDP_BACKUP_AAAA-MM-DD_HH-MM-SS.jsonDocumentos, usuários, contadores e PDFs (codificados em base64) — legível, usado também para sincronizar entre instalações (6.4)
Banco de dados (.db)DB_SGDP_BACKUP_AAAA-MM-DD_HH-MM-SS.dbCópia binária completa do banco SQLite — restauração mais direta, mas só entre instalações do mesmo sistema

Os dois são gerados automaticamente (ao final de cada sessão de uso e periodicamente em segundo plano) e também podem ser gerados manualmente a qualquer momento, pelos botões Exportar backup de cada card. Para restaurar, use o botão Restaurar backup correspondente e selecione o arquivo (.json ou .db).

O botão 📊 Relatório de Backup e Integridade (apenas administradores) mostra um resumo do estado atual: quantos backups existem, tamanhos em disco, data do último backup e contagens gerais do sistema.

Atenção: restaurar qualquer um dos dois tipos de backup apaga os dados atuais e os substitui pelos do arquivo escolhido. Execute essa operação apenas em caso de necessidade — o sistema cria automaticamente um backup do estado atual antes de restaurar, mas ainda assim é uma operação sem volta fácil.
Recomendação: além dos backups automáticos, exporte uma cópia manual periodicamente e armazene em local externo ao computador servidor — servidor de rede, nuvem institucional ou mídia externa.

6.4 Sincronização de backup entre agentes

Quando mais de uma instalação do SGDP é usada (ex: computadores diferentes que não compartilham o mesmo banco), o botão ↻ Sincronizar backup mescla o conteúdo de um backup .json exportado em outra máquina com os dados atuais, em vez de substituir tudo.

  1. Clique em Sincronizar backup e selecione o arquivo .json exportado na outra instalação
  2. O sistema mostra quantos documentos são novos e quantos foram alterados nos dois lados
  3. Documentos alterados em ambos os lados são revisados um a um — escolha "Manter atual" ou "Usar backup" para cada
  4. Confirme para aplicar; um backup de segurança do banco atual é criado automaticamente antes
Como funciona o casamento de documentos: a sincronização identifica o mesmo documento em instalações diferentes pelo conjunto tipo + número + ano (ex: "Lei 12/2026"), nunca pelo identificador interno do banco — cada instalação numera seus próprios registros de forma independente.

6.5 Personalização (tema, fonte, cor, brasão, sons)

Na aba Interface de Configurações:

  • Tema de aparência: claro ou escuro, aplicado imediatamente
  • Largura do conteúdo: compacta (margens laterais confortáveis) ou expandida (aproveita toda a tela útil)
  • Cor de destaque: institucional (padrão), azul, verde ou roxo — muda a cor de botões, links e destaques em todo o sistema
  • Tamanho da fonte: afeta todo o sistema; útil para monitores menores ou acessibilidade
  • Brasão do município: upload de imagem exibida no topo da barra lateral para todos os procuradores (armazenado no servidor, não apenas no navegador de quem enviou)

Essas preferências são individuais — cada procurador escolhe as suas, e ficam salvas no navegador de cada um (exceto o brasão, que é do sistema).

O sistema também toca sons de notificação distintos para clique, sucesso, erro e alertas — não é configurável, mas pode ser silenciado no volume geral do sistema operacional.

6.6 Configuração de e-mail (SMTP)

Para habilitar o botão ✉ Enviar por e-mail (ver 4.3), configure um servidor SMTP na aba Segurança de Configurações: servidor, porta, usuário, senha e remetente. Sem essa configuração, o botão de envio exibe uma mensagem de erro orientando a configurá-lo.

6.7 Diagnóstico de rede

A aba Diagnóstico em Configurações verifica a integridade dos dados (contagem de documentos, arquivos órfãos no disco ou no banco, consistência dos contadores de numeração, total de usuários e último backup). Para diagnosticar problemas de rede (IP local, porta 3001, regras de firewall), execute Diagnostico SGDP.bat diretamente, ou escolha a opção [1] Diagnóstico no menu que aparece ao abrir o Iniciar SGDP.bat — útil quando outro computador não consegue acessar o sistema pela rede.

7. Relatório Gerencial

Acesse Relatório na barra lateral para acompanhar a produção de documentos ao longo do tempo — útil para prestação de contas e planejamento da Procuradoria.

  • Filtros de período: esta semana, este mês, últimos 3 meses, este ano, todo o período ou personalizado (data inicial/final)
  • Total de documentos no período selecionado, com contagem por tipo (Lei, Decreto, Portaria, Parecer, Ofício)
  • Evolução mensal: gráfico de barras mostrando a quantidade de documentos produzidos mês a mês
  • Por assunto: tabela com a quantidade e o percentual de documentos em cada categoria de Assunto
  • Lista de documentos no período, com tipo, número, data, assunto e ementa

Clique em 🖨 Imprimir para gerar uma versão impressa ou PDF do relatório exibido.

O botão 🔁 Comparar com período anterior mostra a variação (▲/▼, diferença e percentual) do total de documentos em relação ao intervalo de mesma duração imediatamente anterior.

Além do relatório gerencial, o sistema gera documentos individuais em papel timbrado (mesmo padrão do órgão, impressos ou exportados em PDF pelo navegador):

  • Pendências e Prazos — lembretes vencidos/a vencer e documentos sem PDF anexado
  • Produtividade por Usuário — ações (criar/editar/upload/assinar) por procurador no período
  • Certidão de Documento (ícone 📜 na listagem) — ficha completa: campos, etiquetas, vínculos, histórico de revisões e status de assinatura
  • Certidão Negativa de Pendências — atesta, a partir da Certidão, a ausência de lembretes pendentes e de PDF faltante para aquele documento (ou lista o que falta, se houver)
  • Certidões em lote — gera as Certidões de todos os documentos do filtro/tipo atual num único PDF
  • Cadeia Normativa (no modal de Vínculos de um documento) — árvore de revogações, alterações, complementos e referências entre documentos
  • Relatório de Etiquetas — documentos agrupados por etiqueta, mais os que não têm nenhuma
  • Backup e Integridade (apenas administradores, em Configurações → Backup) — estado dos backups, tamanhos e contagens do sistema

8. Agenda

A tela Agenda permite cadastrar lembretes com prazo, opcionalmente vinculados a um documento específico — por exemplo, revisar uma Portaria em uma data futura.

  1. Clique em + Novo Lembrete
  2. Informe o título e o prazo
  3. Salve — o lembrete aparece na lista ordenado por prazo

Marque a caixa de seleção ao lado do lembrete para indicá-lo como concluído. A barra lateral exibe um contador com a quantidade de lembretes pendentes já vencidos.

Notificação por e-mail (requer SMTP configurado — ver 6.6): quando um lembrete vence, o sistema envia um e-mail automático para quem o criou (ou para o e-mail padrão, se o criador não tiver e-mail cadastrado). Além disso, um resumo diário com todos os lembretes vencidos e a vencer em breve é enviado uma vez por dia para o e-mail padrão — mesmo que ninguém esteja logado no momento.

9. Lixeira

Ao excluir um documento na listagem, ele não é apagado imediatamente — é enviado para a Lixeira, acessível pela barra lateral, onde fica disponível por 30 dias antes da remoção definitiva e automática.

  • ↺ Restaurar: devolve o documento para a listagem normal, como se nunca tivesse sido excluído
  • ✕ Excluir de vez: remove o documento e o PDF anexado permanentemente, sem esperar os 30 dias — ação irreversível
Atenção: a exclusão definitiva (seja pelo botão "Excluir de vez" ou pela purga automática após 30 dias) não pode ser desfeita, exceto restaurando um backup anterior.

10. Fluxo de Trabalho Recomendado

O fluxo típico para cada ato jurídico:

  1. Elaboração: o procurador redige o documento no Word, utilizando o modelo do tipo correspondente (Lei, Decreto, Portaria etc.)
  2. Numeração prévia (opcional): para inserir o número no corpo do documento antes da assinatura, acesse o SGDP, clique em + Novo [Tipo] e verifique o próximo número disponível no campo "Número" — ele pode ser anotado e usado no Word
  3. Exportação para PDF: no Word, utilize "Salvar como" → PDF ou "Exportar" → PDF
  4. Assinatura digital: assine o PDF pela plataforma de assinatura do município (GOV.BR, DocuSign municipal etc.)
  5. Cadastro no SGDP: acesse a lista do tipo correspondente, clique em + Novo, preencha os campos e faça o upload do PDF assinado
  6. Distribuição: compartilhe o PDF a partir do botão de download do SGDP ou diretamente pelo e-mail/sistema de protocolo

11. Perguntas Frequentes

O servidor não abre ao clicar no .bat

Verifique se o Python está instalado: abra o Prompt de Comando, digite python --version e pressione Enter. Se aparecer um erro, instale o Python em python.org marcando a opção "Add Python to PATH".

A janela do servidor fecha imediatamente

Provavelmente outra instância do SGDP já está em execução na porta 3001, ou outra aplicação ocupa essa porta. Feche todos os processos Python no Gerenciador de Tarefas e tente novamente.

Um procurador não consegue acessar pelo IP

Verifique se: (1) o servidor está em execução no computador correto; (2) ambos os computadores estão na mesma rede Wi-Fi ou cabeada; (3) o firewall do Windows permite conexões na porta 3001 (Painel de Controle → Firewall → Regras de Entrada → Nova Regra → Porta 3001 TCP).

Esqueci a senha de um usuário

Um administrador pode redefinir a senha de qualquer usuário em Usuários → Editar, informando uma nova senha no campo correspondente (deixar em branco mantém a senha atual).

Fechei sem querer a janela do servidor

O servidor só encerra se a janela do terminal for fechada (ou com Ctrl+C) — fechar o navegador não afeta o servidor. Se a janela do terminal foi fechada por engano, basta clicar novamente em Iniciar SGDP.bat para reiniciar — todos os dados são preservados, já que ficam salvos em disco (sgdp.db), não na memória do processo.

Como faço para acessar o sistema fora da rede municipal?

O SGDP é projetado para uso em rede local (intranet). Para acesso externo, seria necessário configurar VPN ou redirecionamento de porta no roteador — o que deve ser avaliado com o setor de TI do município, considerando os aspectos de segurança.

Suporte técnico: em caso de dúvidas ou problemas não cobertos neste manual, entre em contato com o responsável pelo desenvolvimento do sistema.
